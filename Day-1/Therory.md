# Day 01 - HTML Basics

## Overview

The first day of the MERN Stack training introduced the fundamentals of HTML (HyperText Markup Language), the standard language used to create and structure web pages. The session focused on understanding how HTML elements are used to organize content such as text, images, videos, tables, and lists. We also learned the basic structure of an HTML document and the importance of writing clean and well-organized code.

The concepts covered during this session form the foundation of front-end web development. A good understanding of HTML is essential before learning CSS for designing web pages and JavaScript for adding interactivity.

---

## Topics Covered

### 1. HTML Document Structure

We learned the basic structure of an HTML document and the role of each section. Every HTML file begins with a document declaration, followed by the root `<html>` element, which contains the `<head>` and `<body>` sections.

**Main Tags**

* `<!DOCTYPE html>` – Declares the HTML version.
* `<html>` – Root element of the webpage.
* `<head>` – Stores metadata, title, and external resources.
* `<title>` – Displays the webpage title in the browser tab.
* `<body>` – Contains all the visible content of the webpage.

---

### 2. Heading Tags

We studied the six heading tags (`<h1>` to `<h6>`) and learned how they help organize content in a logical order. The `<h1>` tag is used for the main heading, while `<h6>` is used for the smallest heading. Proper use of headings improves readability and makes webpages easier to understand.

---

### 3. Semantic and Non-Semantic Elements

The session explained the difference between semantic and non-semantic HTML elements.

**Semantic Elements**

These tags clearly describe the purpose of the content.

* `<header>`
* `<nav>`
* `<section>`
* `<article>`
* `<footer>`

**Non-Semantic Elements**

These tags do not describe the content but are mainly used for grouping and styling.

* `<div>`
* `<span>`

Using semantic elements improves the structure of a webpage and makes it more accessible.

---

### 4. Block-Level and Inline Elements

We learned how different HTML elements are displayed on a webpage.

**Block-Level Elements**

These elements take up the full width available and always start on a new line.

* `<div>`
* `<p>`
* `<h1>`
* `<section>`

**Inline Elements**

These elements only take up the space required by their content and remain on the same line.

* `<span>`
* `<a>`
* `<strong>`
* `<img>`

Understanding this difference is important when designing webpage layouts.

---

### 5. Subscript and Superscript

We learned how to display text below or above the normal text line using:

* `<sub>`
* `<sup>`

These tags are commonly used in mathematical equations, scientific notations, and chemical formulas such as H₂O and x².

---

### 6. Multimedia in HTML

The session introduced HTML elements used to add multimedia content to webpages.

* `<img>` – Displays images.
* `<audio>` – Plays audio files.
* `<video>` – Embeds video files.

We also learned how to use attributes such as `src`, `controls`, `width`, and `height`.

---

### 7. IFrame and Favicon

We learned how to embed external webpages, maps, or videos using the `<iframe>` element.

The `<link>` tag inside the `<head>` section was used to add a favicon, which appears as the small icon in the browser tab and helps identify the website.

---

### 8. HTML Lists

Different types of lists were studied for organizing information.

* Ordered List (`<ol>`) – Displays numbered items.
* Unordered List (`<ul>`) – Displays bullet points.
* Description List (`<dl>`) – Used for terms and their descriptions.

Lists make information more organized and easier to read.

---

### 9. HTML Tables

We learned how to create tables for displaying data in rows and columns.

The following tags were used:

* `<table>`
* `<tr>`
* `<th>`
* `<td>`

Tables are useful for presenting structured information such as marks, schedules, and reports.
